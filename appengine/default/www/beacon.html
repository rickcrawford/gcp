What good is autocomplete if you can't record the user's interaction with your catalog. Traditional tools like Google Analytics are great for client side interactions, but when it comes to analysis of client interactions at the Edge, you need to look at a Real User Monitoring solution (RUM) for capturing these events. Just looking at server logs isn't enough because Content Delivery Networks (CDNs) will cache your responses so you only see part of the picture.

This project uses Google Cloud Functions to capture every request using a 1x1 gif. The request is then pushed onto Google PubSub for analysis offline in Google Big Query. This provides us the ability to instrument events like - Search, Select, View. The end results of these interactions can then be fed back into our project.

The source for this project is available here: https://github.com/rickcrawford/gcp/tree/master/cloudfunctions/beacon
